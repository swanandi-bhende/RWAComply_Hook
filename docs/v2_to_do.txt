1. Deploy Mock ERC20 Tokens
   Create and deploy two ERC20 tokens that will be used to form a liquidity pool (Token0 and Token1).

2. Approve Token Spending
   Approve the PoolManager (or router later) to spend your deployed tokens from your wallet.

3. Construct PoolKey
   Define a PoolKey struct including:

* token0 address
* token1 address
* fee
* tick spacing
* hook address (your RWAComplyHook)

4. Initialize the Pool
   Call PoolManager.initialize() with the PoolKey and initial price (sqrtPriceX96).

5. Add Liquidity
   Call modifyLiquidity via PoolManager to add initial liquidity to the pool.
   Ensure your hook’s beforeAddLiquidity is triggered and working.

6. Verify Hook Permissions
   Ensure your hook address has correct permission bits set (important for Uniswap v4 hooks to activate).

7. Execute a Swap
   Call swap() on PoolManager with proper SwapParams.
   This should trigger:

* beforeSwap (tier check + revert if needed)
* afterSwap (event emission)

8. Test Compliance Logic

* Try swap with tier = 0 → should revert
* Try swap with tier = 1 or 2 → should pass
