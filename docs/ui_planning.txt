🎨 Visual/UX Principles
Contract-Centric Code Snippets:

On each page, show relevant contract code snippet
Highlight the specific function being demonstrated
Example: On Page 2, show beforeSwap() with tier check highlighted
Transparency via Live Reads:

Every static value should be fetched from contracts, not hardcoded
Display what's actually on-chain
Error-Driven Teaching:

Don't hide errors; use them to teach
Examples: "Tier 0 gets AccessDenied()" shown as educational feature
Judicial Appeal:

Each page has a "Why This Matters for Uniswap" callout box
Example: Page 1 → "Enables KYC/AML-compliant token swaps on Uniswap"
Example: Page 3 → "Hook plugs directly into PoolManager for regulatory pricing"
No Fake Data:

Remove all mock transaction simulation
All interactions are real contract calls
Build fails clearly if deployment is stale
